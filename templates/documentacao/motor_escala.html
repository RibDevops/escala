1. Lógica Operacional Humana

Como o escalante fazia manualmente na planilha Excel — o sistema imita exatamente esse processo.

A planilha do escalante

O escalante trabalhava com uma planilha onde cada linha era um militar e cada coluna era a sequência de serviços. Ao chegar um novo dia de serviço, seguia este raciocínio:

📊
Olha a planilha inteira
Identifica qual militar tem menos serviços no ano — essa é a menor coluna.
⬆️
Varre de baixo para cima
Começa pelos mais modernos (base da lista) e sobe em direção aos mais antigos.
✅
Primeiro disponível escala
Pergunta: "está de férias? Serviu recentemente?" — o primeiro "não" recebe o serviço.
🔄
Esgota a coluna mínima ANTES de avançar
Nunca pula para count maior enquanto houver alguém com count menor disponível.
⚠️
Folga pode ceder no limite
Se toda a menor coluna estiver em folga, quebra a folga do primeiro e registra alerta.
🚫
Férias nunca se quebram
Indisponibilidade real (férias, licença) é absoluta — nem no pior caso.
Sem memória entre dias. A cada novo dia, o escalante olhava a planilha do zero. Não havia anotação de "quem foi o último". A menor coluna era sempre recalculada na hora.

2. Regras do Algoritmo

Declarações explícitas: Não existe fila persistente · Não existe ponteiro entre dias · Não existe round-robin · Tudo é recalculado a cada serviço · A menor coluna é soberana · A varredura é sempre BASE → TOPO · A indisponibilidade NUNCA é quebrada · A folga SÓ pode ser quebrada no fallback quando toda a menor coluna falhar.
1
Limpeza e reversão de quadrinhos

Apaga todos os itens existentes da escala e desconta 1 de cada Quadrinho.quantidade correspondente. A nova geração parte de um estado limpo e consistente.

2
Carregar militares por ordem de antiguidade

Carregados por: posto.ordem_hierarquica ASC → data_ultima_promocao ASC → nome_guerra ASC

  • Índice 0 = TOPO (mais antigo — Ten, Cel...)
  • Índice n-1 = BASE (mais moderno — Cb, Sd...)

Chave de desempate: desempate[m] = (n-1) - índice[m] — BASE recebe 0 (maior prioridade), TOPO recebe n-1 (menor prioridade).

3
Carregar matriz histórica (count global)

Cada militar recebe um count inicial: count[m] = Σ Quadrinho.total somando todos os tipos de serviço (Preto + Vermelho + Roxo = um único número).

4
Carregar indisponibilidades reais

Férias, licença, missão e afastamentos são carregados. Incluem bloqueios pré e pós-férias (se configurados). Estas datas nunca são quebradas.

5
Carry-over inter-mês (folga herdada)

Se o militar serviu nos últimos dias do mês anterior dentro da janela de folga, os primeiros dias do novo mês ficam bloqueados. A folga é global — qualquer tipo de serviço do mês anterior bloqueia todos os tipos.

6
Ordem de processamento: tipo por tipo

Os tipos de serviço são processados por TipoServico.ordem ASC: Preto (todos os 31 dias) → Vermelho → Roxo. A folga criada pelo Preto do dia 28 já estará ativa quando o Vermelho do dia 29 for processado.

7
Processar cada dia — núcleo do algoritmo
  • Ordenação (do zero a cada dia): sorted(militares, key=(count_operacional, desempate))
  • Varredura: percorre em ordem; verifica indisponibilidade (pula), verifica folga (pula), escala o primeiro disponível
  • Toda coluna mínima esgotada antes de tocar qualquer militar com count maior
  • Fallback: só folga bloqueando → quebra folga com alerta · indisponibilidade real → dia vazio
8
Registrar serviço

Banco (persiste): EscalaItem + Quadrinho.quantidade += 1

Só em memória: counts_operacional[m] += 1 · folga_global[m] bloqueia os próximos duração + folga_mínima dias

4. Pseudocódigo Completo

Reflete exatamente o comportamento de escalas/services.py — MotorEscalaVertical.

# ───────────────────────────────────────────────────────────────────────── # MOTOR ESCALA VERTICAL — Pseudocódigo completo # Implementado em: escalas/services.py — class MotorEscalaVertical # ───────────────────────────────────────────────────────────────────────── FUNÇÃO gerar_escala(escala): # ── FASE 1: PREPARAÇÃO ──────────────────────────────────────────────── # Passo 1: Limpeza para cada item em escala.itens: Quadrinho[item.militar][item.tipo_servico].quantidade -= 1 deletar todos os itens da escala # Passo 2: Militares em ordem de antiguidade militares = ORDER_BY( posto.ordem_hierarquica ASC, data_ultima_promocao ASC, nome_guerra ASC ) n = len(militares) índice[m] = posição na lista # 0=TOPO, n-1=BASE desempate[m] = (n - 1) - índice[m] # BASE(n-1) → desempate=0 (MAIOR prioridade no empate) # TOPO(0) → desempate=n-1 (MENOR prioridade no empate) # Passo 3: Matriz histórica — count GLOBAL (todos os tipos somados) counts[m] = 0 para todos para cada q em Quadrinho onde tipo_escala=atual E ano=atual: counts[q.militar] += q.total # soma Preto + Vermelho + Roxo counts_operacional = cópia(counts) # atualizado durante a geração folga_global[m] = conjunto_vazio para todos # Passo 4: Indisponibilidades reais (NUNCA quebradas) indisponibilidades = carregar_indisponibilidades(militares, período) # Passo 5: Carry-over inter-mês janela = duracao_servico + folga_minima # padrão: 1 + 2 = 3 dias para cada servico em mês_anterior onde data >= dia_01 - janela: primeiro_dia_bloqueado = max(dia_01, servico.data + 1) último_dia_bloqueado = servico.data + janela folga_global[servico.militar] += dias entre os dois limites # ── FASE 2: GERAÇÃO ─────────────────────────────────────────────────── # Passo 6: Processar tipo por tipo para cada tipo_servico em ORDER_BY(tipo_servico.ordem ASC): para cada dia em dias_do_tipo em ORDER_BY(data ASC): # Passo 7: Ordenação — recalculada do zero a cada dia militares_ordenados = sorted(militares, chave = (counts_operacional[m], desempate[m]) ) escalado = NENHUM para cada m em militares_ordenados: se dia.data em indisponibilidades[m]: continuar # NUNCA quebra se dia.data em folga_global[m]: continuar # pode ser quebrada no fallback escalado = m parar se escalado != NENHUM: registrar_serviço(dia, escalado) continuar # Fallback bloqueados_só_por_folga = [ m para m em militares_ordenados se dia.data NÃO em indisponibilidades[m] E dia.data em folga_global[m] ] se bloqueados_só_por_folga não vazio: escolhido = bloqueados_só_por_folga[0] gerar_alerta("folga relaxada → " + escolhido.nome) registrar_serviço(dia, escolhido, forcar_escala=True) senão: gerar_alerta_crítico("dia sem cobertura") # ── AUXILIAR: registrar_serviço ──────────────────────────────────────── FUNÇÃO registrar_serviço(dia, militar, forcar_escala=False): EscalaItem.criar(escala, militar, dia, forcar_escala) Quadrinho[militar][dia.tipo_servico].quantidade += 1 counts_operacional[militar] += 1 para k de 1 até janela inclusive: folga_global[militar].adicionar(dia.data + k dias)

5. Garantias do Sistema

🏆
Menor coluna sempre prioritáriaGarantida pela ordenação: todos count=N aparecem antes de qualquer count=N+1.
⬆️
Varredura sempre BASE→TOPOGarantida pelo desempate invertido: índice maior = valor menor = prioridade maior.
✅
Coluna mínima esgotada antes da próximaGarantida: a ordenação agrupa counts iguais consecutivamente.
🚫
Indisponibilidade nunca quebradaGarantida: excluída inclusive do fallback de último recurso.
⚡
Folga só quebrada quando toda coluna falhaGarantida: o fallback classifica o motivo antes de relaxar qualquer bloqueio.
🔢
Count é global (todos os tipos somados)Garantida: _carregar_matriz_historica soma Preto+Vermelho+Roxo.
🌐
Folga global entre tiposGarantida: folga_global é um único dicionário compartilhado.
🔄
Sem ponteiro, sem fila, sem round-robinGarantida: sorted() reexecutado do zero a cada dia.
↩️
Retorno automático após ausênciaGarantida: count não sobe durante férias; o militar volta ao topo naturalmente.
