1. Lógica Operacional Humana

Como o escalante fazia manualmente na planilha Excel — o sistema imita exatamente esse processo.

A planilha do escalante

O escalante trabalhava com uma planilha onde cada linha era um militar e cada coluna era a sequência de serviços realizados. Ao chegar um novo dia de serviço, ele seguia este raciocínio:

📊
Olha a planilha inteira
Identifica qual militar tem menos serviços no ano — essa é a menor coluna.
⬆️
Varre de baixo para cima
Começa pelos mais modernos (base da lista) e sobe em direção aos mais antigos.
✅
Primeiro disponível escala
Pergunta: "está de férias? Serviu recentemente?" — o primeiro "não" recebe o serviço.
🔄
Esgota a coluna mínima ANTES de avançar
Nunca pula para count maior enquanto houver alguém com count menor disponível.
⚠️
Folga pode ceder no limite
Se toda a menor coluna estiver em folga, quebra a folga do primeiro e registra alerta.
🚫
Férias nunca se quebram
Indisponibilidade real (férias, licença) é absoluta — nem no pior caso.
Sem memória entre dias. A cada novo dia, o escalante olhava a planilha do zero. Não havia anotação de "quem foi o último". A menor coluna era sempre recalculada na hora.

2. Regras do Algoritmo

Declarações explícitas: Não existe fila persistente · Não existe ponteiro entre dias · Não existe round-robin · Tudo é recalculado a cada serviço · A menor coluna é soberana · A varredura é sempre BASE → TOPO · A indisponibilidade NUNCA é quebrada · A folga SÓ pode ser quebrada no fallback quando toda a menor coluna falhar.
1
Limpeza e reversão de quadrinhos

Apaga todos os itens existentes da escala e desconta 1 de cada Quadrinho.quantidade correspondente. A nova geração parte de um estado limpo e consistente.

2
Carregar militares por ordem de antiguidade

Carregados por: posto.ordem_hierarquica ASC → data_ultima_promocao ASC → nome_guerra ASC

  • Índice 0 = TOPO (mais antigo — Ten, Cel...)
  • Índice n-1 = BASE (mais moderno — Cb, Sd...)

Chave de desempate calculada automaticamente: desempate[m] = (n-1) - índice[m] — BASE recebe 0 (maior prioridade), TOPO recebe n-1 (menor prioridade).

3
Carregar matriz histórica (count global)

Cada militar recebe um count inicial carregado do banco:

  • Fonte: tabela Quadrinho
  • Filtro: mesmo tipo_escala + mesmo ano
  • Soma TODOS os tipos de serviço (Preto + Vermelho + Roxo = um único número)
  • Fórmula: count[m] = Σ Quadrinho.total para todos os tipos
4
Carregar indisponibilidades reais

Férias, licença, missão e afastamentos são carregados. Opcionalmente incluem bloqueios pré e pós-férias. Estas datas nunca são quebradas em nenhuma circunstância.

5
Carry-over inter-mês (folga herdada)

Se o militar serviu nos últimos dias do mês anterior dentro da janela de folga, os primeiros dias do novo mês ficam bloqueados. A folga é global — qualquer tipo de serviço (Preto ou Vermelho) do mês anterior bloqueia todos os tipos.

6
Ordem de processamento: tipo por tipo

Os tipos de serviço são processados por TipoServico.ordem ASC:

  • Preto (ordem=0): todos os 31 dias gerados completamente
  • Vermelho (ordem=1): todos os 31 dias gerados completamente
  • Roxo (ordem=2): etc.

A folga criada pelo Preto do dia 28 já estará ativa quando o Vermelho do dia 29 for processado — ambos compartilham o mesmo folga_global.

7
Processar cada dia — núcleo do algoritmo

Para cada dia de cada tipo:

  • Ordenação (do zero a cada dia): sorted(militares, key=(count_operacional, desempate)) — menor count primeiro, desempate BASE→TOPO
  • Varredura principal: percorre a lista em ordem; verifica indisponibilidade (pula se sim), verifica folga (pula se sim), escala o primeiro disponível
  • Toda a menor coluna é esgotada antes de tocar qualquer military com count maior — garantido pela ordenação
  • Fallback: se toda a varredura falhou, classifica os motivos (só folga → quebra folga com alerta; indisponibilidade real → dia vazio)
8
Registrar serviço

Persiste no banco: EscalaItem + Quadrinho.quantidade += 1

Só em memória (descartado ao fim): counts_operacional[m] += 1 · folga_global[m] recebe os próximos duração + folga_mínima dias bloqueados

── FASE 1: PREPARAÇÃO ──────────────────────────────────────────────── # Passo 1: Limpeza para cada item em escala.itens: Quadrinho[item.militar][item.tipo_servico].quantidade -= 1 deletar todos os itens da escala # Passo 2: Militares em ordem de antiguidade militares = ORDER_BY( posto.ordem_hierarquica ASC, data_ultima_promocao ASC, nome_guerra ASC ) n = len(militares) índice[m] = posição na lista # 0=TOPO, n-1=BASE desempate[m] = (n - 1) - índice[m] # ── FASE 2: GERAÇÃO ─────────────────────────────────────────────────── # Passo 6: Processar tipo por tipo, em ordem (Preto completo → Vermelho...) para cada tipo_servico em ORDER_BY(tipo_servico.ordem ASC): para cada dia em dias_do_tipo em ORDER_BY(data ASC): # Passo 7: Ordenação — recalculada do zero a cada dia militares_ordenados = sorted(militares, chave = (counts_operacional[m], desempate[m]) ) # Toda a menor coluna aparece ANTES de qualquer coluna maior. # Empate resolvido por desempate: menor = BASE = mais moderno = prioridade. # Varredura principal escalado = NENHUM para cada m em militares_ordenados: se dia.data em indisponibilidades[m]: log("indisponibilidade") continuar # pula — NUNCA quebra se dia.data em folga_global[m]: log("folga") continuar # pula — pode ser quebrada no fallback escalado = m parar se escalado != NENHUM: registrar_serviço(dia, escalado) continuar # próximo dia # Fallback: toda a varredura principal falhou bloqueados_só_por_folga = [ m para m em militares_ordenados se dia.data NÃO em indisponibilidades[m] E dia.data em folga_global[m] ] se bloqueados_só_por_folga não vazio: escolhido = bloqueados_só_por_folga[0] # 5. Garantias do Sistema

🏆
Menor coluna sempre prioritáriaGarantida pela ordenação: todos os count=N aparecem antes de qualquer count=N+1.
⬆️
Varredura sempre BASE→TOPOGarantida pelo desempate invertido: índice maior = valor menor = prioridade maior.
✅
Toda coluna mínima esgotada antes da próximaGarantida: a ordenação agrupa counts iguais consecutivamente.
🚫
Indisponibilidade nunca quebradaGarantida: excluída inclusive do fallback de último recurso.
⚡
Folga só quebrada quando toda coluna falhaGarantida: o fallback classifica o motivo antes de relaxar qualquer bloqueio.
🔢
Count é global (todos os tipos somados)Garantida: _carregar_matriz_historica soma Preto+Vermelho+Roxo.
🌐
Folga global entre tiposGarantida: folga_global é um único dicionário compartilhado entre Preto, Vermelho e Roxo.
🔄
Sem ponteiro, sem fila, sem round-robinGarantida: sorted() é reexecutado do zero a cada dia.
↩️
Retorno automático após ausênciaGarantida: count não sobe durante férias; o militar volta ao topo naturalmente.
